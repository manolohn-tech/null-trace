🖼 How Image Steganography Works
1
Upload a Carrier Image
Choose any PNG, JPG, or BMP image. Larger images offer more hidden capacity — a 1920×1080 image at 1-bit depth can store roughly 750 KB of encrypted data. The image itself is never altered visibly.
TIP: Use PNG for best results — JPG compression destroys hidden data
2
Write Your Secret Message & Set a Password
Type the message you want to hide. Enter any password — it's used with PBKDF2 (100,000 rounds) to derive a strong AES-256 encryption key. Use "Generate" for a cryptographically random 36-character key.
3
Choose LSB Depth (1–4 bits)
Controls how many least-significant bits per colour channel are used. Depth 1 is invisible to the human eye. Depth 4 gives 4× the capacity but may introduce a slight colour shift — not recommended for sensitive images.
RULE: Note your depth — the receiver must use the exact same value to decode
4
Encode & Download the Stego PNG
Your message is AES-256-GCM encrypted, then the encrypted bytes are scattered across the image's pixel colour values by overwriting the lowest bits. The output is a lossless PNG that looks identical to the original.
5
Decode on the Other End
Load the exact downloaded PNG into the Decode panel. Use the same password and the same LSB depth. The tool extracts the hidden bits, reassembles the ciphertext, and decrypts your message.
WARNING: Never re-save, compress, or screenshot the PNG — this destroys the hidden data
Security note: All processing happens entirely in your browser. No data is ever uploaded or sent anywhere. The encryption uses the Web Crypto API's native AES-256-GCM — the same standard used by banks and governments.
🎵 How Audio Steganography Works
1
Upload a WAV Audio File
Only uncompressed WAV files work — formats like MP3 use lossy compression that destroys hidden data. A 1-minute stereo WAV at 44.1 kHz gives you roughly 660 KB of hidden storage.
TIP: Use longer WAV files for more capacity
2
Type Your Message & Password
Enter your secret message and a strong password. The password goes through PBKDF2 key derivation with 100,000 iterations before being used to AES-256-GCM encrypt your message.
3
LSB Injection into Audio Samples
The encrypted bytes are converted to bits. Each bit overwrites the least-significant bit of a PCM audio sample. At 16-bit audio, flipping the last bit changes a sample's value by only 1 out of 65,536 — completely inaudible.
The waveform visualiser shows your audio — after encoding it looks identical
4
Download the Stego WAV
The output is a WAV file that sounds exactly like the original. Share it like any audio file — via email, messaging apps, or file transfer. Do not convert it to MP3 or any other format.
5
Extract on the Other End
Load the stego WAV into the Decode panel. Enter the same password. The tool reads the LSBs of each sample, reassembles the encrypted payload, and decrypts your original message.
WARNING: Only share and use the exact downloaded WAV — never re-encode or convert it
Security note: The human ear cannot detect changes smaller than 1/100th of a percent in amplitude. A 1-bit change to a 16-bit sample is 0.0015% — well below any audible threshold, even on studio-quality headphones.
📄 How Text Steganography Works
1
Write Cover Text & Your Secret Message
The cover text is what everyone sees — write something natural and plausible for your context, like a social media post, email, or article excerpt. Your secret message will be hidden inside it, completely invisibly.
2
Choose an Encoding Method
Three methods are available. Zero-Width Characters inserts invisible Unicode control characters between letters — undetectable in almost all platforms. Trailing Whitespace appends spaces or tabs to each line. Unicode Homoglyphs swaps select Latin letters for visually identical Cyrillic ones.
RECOMMENDED: Zero-Width Characters — invisible on every major platform
3
Encrypt & Embed
Your secret message is AES-256-GCM encrypted using your password, then the encrypted bytes are converted to bits. Each bit maps to one invisible character (or whitespace, or homoglyph) inserted into the cover text.
4
Copy & Send the Stego Text
The output looks exactly like your original cover text. Copy it and send it anywhere — messages, emails, social posts. The hidden data travels with it invisibly. Make sure to copy the entire text, including any invisible characters.
TIP: Use platforms that preserve Unicode — avoid apps that strip invisible characters
5
Decode by Pasting the Text
Paste the stego text into the Decode panel, enter the same password, and select the same method. The tool extracts the hidden bits, decrypts the payload, and reveals your original message.
WARNING: Both sender and receiver must use the exact same encoding method
Capacity note: Text steganography has much lower capacity than image or audio methods. Each cover text character can carry approximately 1 hidden bit. A 500-word cover text can hide roughly 60 characters of plaintext after encryption overhead.
